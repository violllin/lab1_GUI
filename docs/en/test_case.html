Parser Syntax Analysis Testing

Below are the testing results of the developed parser for Swift lambda expressions using both correct and erroneous data.

1. Test Case 1 (Correct Expression)
let calc = { (a: Int, b: Int, c: Int) -> Int in return a + (b * c) };

Result: Analysis completed successfully, no errors found.

Fig. 1 — Test Case 1

2. Test Case 2 (Correct Expression)
let simple = { (s: String) -> String in return s };

Result: Analysis completed successfully, no errors found.

Fig. 2 — Test Case 2

3. Test Case 3 (Errors in Types and Punctuation)
let calc = { (a: I%nt, b:::: Int c: Int,,,,) -> Int in return a + (b * c) };

Result: Errors in type declarations and redundant punctuation marks were detected and neutralized.

Fig. 3 — Test Case 3

4. Test Case 4 (Multiple Syntax Errors)
!!let calc = { (: Int, b: Int c: Int,,) -@> Int in @@ return a + (b * c)};;;

Result: Errors identified in keywords, missing identifiers, and incorrect operators.

Fig. 4 — Test Case 4

5. Test Case 5 (Errors in Identifiers and Operators)
?let c^alc = { (a: Int, b: Int, c: Int) -?> Int i+n return a + (b * c) };

Result: Neutralization of errors in variable names and reserved keywords (in).

Fig. 5 — Test Case 5

6. Test Case 6 (Use of Special Characters)
let @ calc = @ { (a: Int, b: @ Int, c: Int) -@> Int in @ return @ a + (b * c)}; @

Result: Diagnostics of redundant '@' characters in various parts of the expression.
