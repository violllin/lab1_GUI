Тестирование синтаксического анализатора

Ниже представлены результаты тестирования разработанного парсера для лямбда-выражений языка Swift на корректных и ошибочных данных.

1. Тестовый пример 1 (Корректное выражение)
let calc = { (a: Int, b: Int, c: Int) -> Int in return a + (b * c) };

Результат: анализ завершен успешно, ошибок не обнаружено.

3. Тестовый пример 3 (Ошибки в типах и знаках препинания)
let calc = { (a: I%nt, b:::: Int c: Int,,,,) -> Int in return a + (b * c) };

Результат: обнаружены и нейтрализованы ошибки в объявлении типов и лишние символы пунктуации.

4. Тестовый пример 4 (Множественные синтаксические ошибки)
!!let calc = { (: Int, b: Int c: Int,,) -@> Int in @@ return a + (b * c)};;;

Результат: выявлены ошибки в ключевых словах, пропусках идентификаторов и некорректных операторах.

5. Тестовый пример 5 (Ошибки в идентификаторах и операторах)
?let c^alc = { (a: Int, b: Int, c: Int) -?> Int i+n return a + (b * c) };

Результат: нейтрализация ошибок в именах переменных и служебных конструкторах (in).
